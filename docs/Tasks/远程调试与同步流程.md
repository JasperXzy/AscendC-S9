# 远程调试与同步流程

本项目本地运行在 Mac 客户端，没有 CANN 环境；CANN 编译、安装、测试和 profiling 需要通过 SSH 到远程 `modelart-s9` 执行。

## 路径约定

| 环境 | 路径 |
|---|---|
| 本地项目 | `/Users/jasperxzy/Projects/AscendC-S9` |
| 远程 SSH | `modelart-s9` |
| 远程工作根目录 | `/home/ma-user/work` |
| 远程项目目录 | `/home/ma-user/work/AscendC-S9` |

默认以本地源码为主。也就是说，正常开发时先在本地修改，再同步到远程编译和测试。

## 标准工作流

1. 在本地阅读和修改源码、文档。
2. 修改后同步到远程：

   ```bash
   rsync -az --delete \
     --exclude '.git/' \
     --exclude '.DS_Store' \
     --exclude 'build_out/' \
     --exclude 'dist/' \
     --exclude 'build/' \
     --exclude 'PROF*/' \
     /Users/jasperxzy/Projects/AscendC-S9/ \
     modelart-s9:/home/ma-user/work/AscendC-S9/
   ```

3. 在远程执行 CANN 编译、安装、公开 case 测试和 profiling。
4. 如果远程调试时临时改了源码，要立刻同步回本地，避免两边代码分叉。
5. 提交前确认本地源码、远程源码、编译 `.run` 包对应的是同一份代码。

## 远程执行示例

进入远程项目：

```bash
ssh modelart-s9
cd /home/ma-user/work/AscendC-S9
```

运行某个公开测试：

```bash
cd case_910b/Greater
bash run.sh 1
```

具体算子工程的 `./build.sh`、`.run` 安装和提交包生成，以当时实现的工程目录为准。

## 同步规则

默认同步源码、脚本、文档和提交所需文件。

默认不同步这些临时产物：

- `build_out/`
- `dist/`
- `build/`
- `PROF*/`
- 缓存目录
- 临时日志

需要提交或留档时，可以单独同步以下产物：

- `custom_opp_*.run`
- 最终提交 `.zip`
- 必要的性能结果摘要

## 注意事项

1. 不要只在远程改源码而不回传本地。
2. 不要只在本地改源码就直接沿用远程旧构建结果。
3. `rsync --delete` 会让远程目录镜像本地目录；执行前确认远程目录没有需要保留但未回传的源码改动。
4. 构建产物和 profiling 结果通常很大且容易污染仓库，默认留在远程即可。
5. 最终提交包中的 `op_host/`、`op_kernel/` 和 `custom_opp_*.run` 必须来自同一份源码。
